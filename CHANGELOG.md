# Changelog

## 0.1.0-alpha5

- Увеличены шрифты, межстрочные интервалы, поля, кнопки и строки таблиц.
- Почти чёрный фон заменён на тёмно-серый, яркий зелёный акцент — на спокойный бирюзовый.
- Убрано лишнее свечение статусов и активных элементов.
- Добавлена проверка готовности к перезагрузке: systemd-автозапуск, awg0.conf, IPv4 forwarding и NAT.
- Добавлен безопасный диагностический отчёт без закрытых ключей и токенов.
- Добавлены uptime web-панели и AWG-сервера.
- Добавлены автоматические ежедневные резервные копии через systemd timer.
- Добавлена защита входа: пять неудачных попыток за 15 минут с одного IP.
- Добавлен необязательный скрипт HTTPS с Nginx и Let’s Encrypt.
- Python-пакет устанавливается в virtualenv, поэтому команда awgpanel работает из любого каталога.

## 0.1.0-alpha4

- Интерфейс полностью приведён к стилю SG-Panel с отдельным бирюзовым акцентом.
- Главная страница стала чистым Overview без форм настройки.
- Добавлены отдельные страницы: Server, Clients, Access, Routing, DNS, Backups, Security и Settings.
- Добавлены персональные ссылки на актуальный клиентский `.conf`, счётчик обращений и пересоздание токена.
- Добавлен выбор `AllowedIPs` для каждого клиента: весь трафик или заданные сети.
- Добавлена серверная изоляция клиентов через firewall-правило `awg0 → awg0`.
- Добавлено изменение DNS из отдельной страницы.
- Добавлены ручное создание и восстановление резервных копий из web-панели.
- Добавлена смена пароля администратора.
- Исправлен updater: обновление установленной панели больше не вызывает `apt`, `dpkg` и полный установщик.
- Исправлено ожидание пакетов: проверяются реальные lock-файлы, а не постоянный процесс `unattended-upgrade-shutdown`.
- Сохранена автоматическая миграция базы Alpha 3 без потери сервера, клиентов и ключей.
- Добавлена подробная документация по каждой странице и режиму.

## 0.1.0-alpha3

- Исправлена проверка временного файла: `awg-quick` получает допустимое имя `awgtest0.conf`.
- Добавлено автоматическое определение публичного IPv4 и внешнего интерфейса.
- Настройка сервера выполняется одной кнопкой с резервной копией и откатом.
- Добавлена отдельная страница Diagnostics.
- Общий процент RAM отделён от реальной памяти процесса web-панели.

## 0.1.0-alpha2

- Исправлена остановка установки на хеше пароля с символами `$`.
- `web.env` больше не подключается как shell-скрипт.
- Проверка Ubuntu выполняется до установки панели.

## 0.1.0-alpha1

- Первый отдельный репозиторий только для AmneziaWG 2.0.
- Настройка AWG 2.0, клиенты, `.conf`, QR, handshake и RX/TX.
