# Changelog

## v0.7.0-RC5 — 2026-07-15

- Controller и до 12 SG-Node получили постоянные непересекающиеся VPN-пулы: `10.77.0.0/24` для Controller и `10.77.1.0/24` … `10.77.12.0/24` для Node.
- При обновлении существующие клиенты SG-Node переносятся в пул своего сервера; Agent одновременно меняет адрес `awg0`, локальные peers и управляемые сетевые правила.
- Номер пула хранится в базе, отображается в Cluster и Clients и не переиспользуется автоматически после удаления Node.
- Обычный клиентский профиль экспортирует точную строку `AllowedIPs = 0.0.0.0/0, ::/0`, необходимую официальному AmneziaVPN для полного туннеля и настройки раздельного туннелирования.
- Пользовательские маршруты и исключения по-прежнему экспортируются как явно заданный набор сетей.
- Резервные копии веб-панели и SSH используют один формат с отдельным `manifest.json`, SHA-256 всех файлов, проверкой metadata, SQLite `integrity_check`, JSON, AWG-конфигурации и безопасных путей.
- Проверка запускается после создания и перед восстановлением; повреждённая или несовместимая копия блокируется до любых изменений системы.
- В копию включены база, ключи и клиенты, `awg0`, настройки панели и Agent, управляемые Nginx/HTTPS-файлы, Traffic Rules и служебное состояние; вся Ubuntu и посторонние файлы не копируются.
- Из SSH-меню удалены дублирующие пункты старых номеров 2, 5, 8 и 12; оставшиеся действия перенумерованы, добавлено отдельное действие проверки резервной копии. `repair-access` сохранён как скрытая аварийная команда.
- `install.sh`, updater и uninstall работают непосредственно с GitHub `main`; Release и тег для установки не требуются.
- Установщик заранее проверяет внутренний TCP `18080`, выбранный публичный TCP-порт и UDP `585`.
- Версии: Panel `0.7.0-RC5`, UI `sgawg070rc5`, Node Agent `0.7.0-RC5`, Python package `0.7.0rc5`.

## v0.7.0-RC4 — 2026-07-15

- Добавлено единое нумерованное SSH-меню `sudo sg-awg-panel` для состояния, диагностики, пароля, сессий, Clients, Cluster, Cascade, backup/restore, обновления и uninstall.
- Пункт 6 меняет пароль, пункт 9 проверяет клиентов, пункт 10 — Cluster/SG-Node, пункт 11 — Cascade.
- Исправлен `CSRF token mismatch` на чистых установках и при открытии нескольких панелей в одном браузере.
- Каждая установка использует собственное имя cookie и постоянный secret; устаревший CSRF безопасно возвращает на страницу входа.
- Сохранена единая модель установки: один полный installer и один updater для любой EC2.
- Исправлена синхронизация реального публичного ключа работающего `awg0` между SG-Node и Controller.
- Controller учитывает фактически занятые `AllowedIPs` Node; Agent блокирует повторяющиеся VPN-адреса.
- Сохранены локальные peers Node и обычный формат профиля удалённого клиента.
- Сохранён классический интерфейс и все исправления Clients, Cluster и Cascade сборок 207–211.
- GitHub Actions использует временные служебные пути вместо `/var/lib/sg-awg-panel`.
- Добавлена структура `docs/screenshots/` для шести основных GitHub-скриншотов RC4.
- Версии: Panel `0.7.0-RC4`, UI `sgawg070rc4`, Node Agent `0.7.0-RC4`.

## v211 — 2026-07-14

## v210 — 2026-07-14

- Возвращён проверенный классический дизайн 208; экспериментальный интерфейс 209 полностью исключён.
- Добавлен отдельный `update.sh`: резервная копия, копирование по checksum, проверка ключевых файлов, health-check и автоматический откат без повторной настройки AWG/Nginx.
- Cluster показывает одну SG-Node на имя даже при старых дублях; новая команда подключения по-прежнему появляется сразу после одной кнопки.
- Clients сокращён до шести полезных колонок без обязательной горизонтальной прокрутки; VPN IP и последняя активность перенесены в строки клиента и сервера.

- Результат проверки клиента Cascade остаётся прямо под кнопкой и честно разделяет серверный выходной IP и метку активного клиентского соединения.
- При Direct ↔ Cascade conntrack очищается по original/reply tuple только для изменённых VPN-адресов; VPN-туннель вручную переподключать не требуется.
- UI build: `sgawg210`.

## v208 — 2026-07-13

- Cluster упрощён до четырёх шагов; одноразовая команда подключения показывается сразу под формой и не исчезает при фоновой проверке статуса.

- Проверка клиента Cascade подтверждает handshake, трафик и правильную route mark активного соединения.
- Таблица Clients стала компактнее: короткие заголовки, нормальные пропорции колонок и меньше лишней горизонтальной прокрутки.
- UI build: `sgawg208`.
