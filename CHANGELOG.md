# Changelog

## 0.1.0-alpha7

- Внешний доступ переведён на Nginx; backend панели всегда слушает только `127.0.0.1:18080`.
- Добавлена настройка HTTP/HTTPS, домена и публичного порта панели.
- Добавлены серверные активные сессии, завершение отдельных сессий и автоматический выход всех устройств после смены пароля.
- Добавлен IP allowlist административной части панели с защитой от блокировки текущего IP.
- Добавлен журнал успешных, ошибочных и заблокированных входов.
- Добавлены расписание резервных копий и срок хранения от 1 до 365 копий.
- Добавлена проверка обновлений по тегам GitHub и запуск фонового обновления из панели.
- Updater сохраняет проект, базу, env, Nginx и systemd units; при ошибке выполняется автоматический откат.
- Добавлена `sg-awg-recovery.service` для восстановления Nginx, панели, IPv4 forwarding и AWG после reboot.
- Diagnostics показывает Nginx, loopback backend и готовность восстановления после перезагрузки.

## 0.1.0-alpha6

- Типографика, ширина бокового меню, рабочие отступы, размеры карточек, полей, кнопок и таблиц приведены к визуальной системе SG-Panel.
- Основной фон сделан тёмно-серым; яркий зелёный акцент заменён на спокойный серо-синий.
- На странице Server добавлены кнопки определения публичного IPv4 и внешнего интерфейса.
- Добавлены проверенный профиль маскировки и генерация новых диапазонов H1–H4.
- Для клиента добавлены отдельные название, комментарий, DNS и MTU.
- В Routing добавлен режим «Только сеть AWG».

## 0.1.0-alpha5

- Увеличена читаемость интерфейса, добавлены Diagnostics, uptime и память процесса панели.
- Добавлены ежедневные резервные копии, защита входа и необязательная установка HTTPS.
- Python-пакет устанавливается в virtualenv, поэтому команда `awgpanel` работает из любого каталога.

## 0.1.0-alpha4

- Интерфейс разделён на Overview, Server, Clients, Access, Routing, DNS, Diagnostics, Backups, Security и Settings.
- Добавлены персональные ссылки на актуальный клиентский `.conf`, счётчик обращений и пересоздание токена.
- Добавлен выбор `AllowedIPs`, серверная изоляция клиентов, резервные копии и смена пароля.
- Исправлен лёгкий updater без `apt`, `dpkg` и перезапуска рабочего AWG.

## 0.1.0-alpha3

- Исправлена проверка временного файла через допустимое имя `awgtest0.conf`.
- Добавлены автоматическое определение публичного IPv4 и интерфейса, Diagnostics и память процесса панели.

## 0.1.0-alpha2

- Исправлена запись хеша пароля и ранняя проверка поддерживаемой версии Ubuntu.

## 0.1.0-alpha1

- Первый отдельный репозиторий только для AmneziaWG 2.0.
- Настройка AWG 2.0, клиенты, `.conf`, QR, handshake и RX/TX.
