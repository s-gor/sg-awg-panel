# Changelog

## 0.1.0-alpha3

- Исправлена проверка временного файла: `awg-quick` получает допустимое имя `awgtest0.conf`.
- Добавлено ожидание завершения `unattended-upgrades` и освобождения блокировок `apt/dpkg`.
- Добавлено автоматическое определение публичного IPv4 и внешнего интерфейса.
- Настройка сервера выполняется одной кнопкой: резервная копия, проверка, сохранение и запуск/перезапуск.
- Добавлен автоматический откат при ошибке применения конфигурации.
- Резервная копия базы и `awg0.conf` создаётся перед изменением сервера или клиентов.
- Добавлено пересоздание ключей существующего клиента без изменения его адреса.
- RX/TX отображаются в читаемом виде, сохранено время последнего handshake.
- Добавлена отдельная страница Diagnostics: службы, UDP-порт, интерфейс, ресурсы, резервные копии и журналы.
- Общий процент RAM отделён от реальной памяти процесса web-панели.
- Добавлены отдельные скрипты обновления из GitHub и полного удаления.
- В интерфейсе явно указано, что длинный конфиг не нужно копировать вручную: используйте `.conf` или QR.

## 0.1.0-alpha2

- Исправлена остановка установки на хеше пароля с символами `$`.
- `web.env` больше не подключается как shell-скрипт.
- Проверка Ubuntu 22.04/24.04 выполняется до установки панели.
- AmneziaWG устанавливается до запуска web-панели.
- Сообщение `Ready` больше не появляется при незавершённой установке.

## 0.1.0-alpha1

- Первый отдельный репозиторий только для AmneziaWG 2.0.
- Полностью отделён от SG-Panel и Xray.
- Отдельные пути, база, службы и установщик.
- Настройка AWG 2.0, клиенты, `.conf`, QR, handshake и RX/TX.
- Мониторинг общей памяти и load average.
- Установка без Docker на Ubuntu 22.04/24.04.
