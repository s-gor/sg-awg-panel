# SG-AWG-Panel

Лёгкая самостоятельная web-панель для **AmneziaWG 2.0**.

Проект не использует Xray и не является модулем SG-Panel. Он управляет собственным AWG-сервером: интерфейсом `awg0`, клиентами, ключами, Routing, DNS, доступом к конфигам, резервными копиями, безопасностью и диагностикой.

Текущая версия: **v0.1.0-alpha7**.

## Что уже работает

- нативная установка без Docker на Ubuntu 22.04/24.04;
- AmneziaWG kernel module, `awg0`, IPv4 forwarding и NAT;
- создание клиентов, `.conf`, QR-коды, handshake и RX/TX;
- полный туннель, только сеть AWG и пользовательские `AllowedIPs`;
- индивидуальные DNS и MTU клиента;
- персональные ссылки на актуальный конфиг;
- Nginx как внешний вход и backend только на `127.0.0.1:18080`;
- выбор публичного порта панели;
- HTTPS и домен через Let’s Encrypt;
- смена пароля с завершением всех сессий;
- активные сессии и их принудительное завершение;
- IP allowlist для административной части панели;
- журнал успешных, ошибочных и заблокированных входов;
- настраиваемое расписание и срок хранения резервных копий;
- проверка новых тегов GitHub из панели;
- фоновое обновление с полной резервной копией и автоматическим откатом;
- recovery service для восстановления панели, Nginx и AWG после перезагрузки;
- диагностический отчёт без ключей и токенов.

Схема работы:

```text
AmneziaVPN / AmneziaWG client
          |
          | UDP 585 / AWG 2.0
          v
      VPS / awg0
          |
          | Linux Routing + NAT
          v
       Интернет
```

Web-панель работает отдельно:

```text
Браузер
   |
   | HTTP или HTTPS / выбранный порт
   v
Nginx
   |
   | 127.0.0.1:18080
   v
SG-AWG-Panel backend
```

## Требования

- Ubuntu 22.04 или 24.04;
- публичный IPv4;
- TCP `22` — только со своего IP;
- TCP `8080` — первоначальный порт панели, только со своего IP;
- UDP `585` — для AWG-клиентов;
- для HTTPS дополнительно TCP `80` и выбранный HTTPS-порт.

## Чистая установка

Под обычным пользователем вставьте одним блоком:

```bash
bash <<'BASH'
set -Eeuo pipefail
tmp="$(mktemp /tmp/sg-awg-install.XXXXXX.sh)"
trap 'rm -f "$tmp"' EXIT
curl -fsSL \
  https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.1.0-alpha7/install-from-github.sh \
  -o "$tmp"
sudo bash "$tmp"
BASH
```

После установки откройте:

```text
http://PUBLIC_IP:8080
```

Nginx принимает внешний запрос, а backend панели уже привязан только к loopback.

## Первый запуск AWG

1. Откройте **Server**.
2. Проверьте публичный IPv4 и внешний интерфейс.
3. Оставьте для первого теста UDP `585`, сеть `10.77.0.0/24`, MTU `1280` и проверенный профиль маскировки.
4. Нажмите **«Сохранить и запустить»**.
5. Откройте **Clients** и создайте устройство.
6. Скачайте `.conf` или используйте QR-код.
7. На Windows импортируйте `.conf` в **AmneziaVPN**.

Обычный WireGuard не понимает параметры AWG 2.0 и для этих конфигов не подходит.

## HTTPS, домен и порт панели

Откройте **Settings → Публичный доступ** и укажите:

- HTTP или HTTPS;
- домен;
- публичный порт;
- e-mail Let’s Encrypt для HTTPS.

Перед выпуском сертификата домен должен указывать на сервер, а TCP `80` должен быть открыт. После перехода на HTTPS старый публичный порт можно закрыть в cloud firewall.

## Безопасность

На странице **Security** доступны:

- смена пароля;
- IP allowlist;
- список активных сессий;
- завершение отдельных или всех остальных сессий;
- журнал входов.

При смене пароля все сессии завершаются автоматически.

## Резервные копии

На странице **Backups** можно выбрать:

- каждый час;
- каждые 6 часов;
- ежедневно;
- раз в неделю;
- отключить расписание;
- хранить от 1 до 365 последних копий.

## Обновление

Проверка и запуск обновления доступны в **Settings**. Обновление выполняется отдельной systemd-задачей. Перед изменениями сохраняются проект, база, `web.env`, Nginx и systemd units. Если новая версия не проходит health-check, предыдущая версия восстанавливается автоматически.

Ручной вариант:

```bash
sudo env SG_AWG_PANEL_VERSION=v0.1.0-alpha7 \
  bash /opt/sg-awg-panel/deploy/update-from-github.sh
```

Рабочий AWG-туннель при обычном обновлении панели не перезапускается.

## Проверка после reboot

```bash
systemctl is-active sg-awg-panel
systemctl is-active nginx
systemctl is-active sg-awg-server
systemctl is-active sg-awg-recovery
ip -br addr show awg0
```

На странице **Diagnostics** проверяются systemd, loopback backend, Nginx, `awg0.conf`, IPv4 forwarding, NAT и recovery service.

## Ограничения Alpha 7

- только IPv4;
- один интерфейс `awg0`;
