# SG-AWG-Panel

**Система развёртывания и обслуживания сервера на базе движка AmneziaWG 2.0 с собственной веб-панелью управления.**

![Version](https://img.shields.io/badge/version-v0.7.0--RC3-3b82f6)
![Ubuntu](https://img.shields.io/badge/Ubuntu-22.04%20%7C%2024.04-E95420?logo=ubuntu&logoColor=white)
![Python](https://img.shields.io/badge/Python-3.10%2B-3776AB?logo=python&logoColor=white)
![AmneziaWG](https://img.shields.io/badge/AmneziaWG-2.0-2563EB)

Универсальные VPN-панели обычно объединяют множество протоколов, транспортов и сценариев. Это расширяет их возможности, но одновременно усложняет настройку, обслуживание и диагностику.

**SG-AWG-Panel решает одну конкретную задачу:** разворачивает на чистом сервере Ubuntu нативный AmneziaWG 2.0 и предоставляет собственную веб-панель для управления сервером, клиентами, Traffic Rules, Cascade, SG-Node, безопасностью, резервными копиями и обновлениями.

Это не просто графическая оболочка над готовой конфигурацией. SG-AWG-Panel устанавливает необходимые компоненты, создаёт и применяет настройки сервера и помогает обслуживать всю установленную систему через единый интерфейс.

```text
Обычный маршрут: Client → AWG Server → Internet
Cascade:        Client → Entry SG-AWG → Exit SG-AWG → Internet
Cluster:        Controller → SG-Node Agent → AmneziaWG runtime
```

Пользовательский трафик проходит через AmneziaWG напрямую. Для работы соединения не требуются Xray, VLESS, Reality или отдельная прокси-цепочка.

> Текущая версия: `v0.7.0-RC3`.

## Что делает SG-AWG-Panel

### AWG Server

- устанавливает AmneziaWG и необходимые системные компоненты;
- создаёт и применяет конфигурацию сервера;
- позволяет изменять Endpoint, UDP-порт, внутреннюю сеть, DNS, MTU и параметры AmneziaWG;
- показывает состояние сервера;
- позволяет запускать, останавливать и перезапускать AWG Server.

### Clients

- создаёт отдельного клиента и пару ключей для каждого устройства;
- назначает клиенту внутренний IP;
- позволяет включать, отключать, изменять и удалять клиентов;
- поддерживает срок действия клиента;
- показывает полный текст конфигурации;
- создаёт QR-код;
- позволяет копировать конфигурацию и скачивать файл `.conf`;
- поддерживает персональную защищённую ссылку на конфигурацию.

### Traffic Rules

Traffic Rules — полноценная рабочая возможность SG-AWG-Panel для гибкой блокировки трафика клиентов.

Панель позволяет:

- создавать и изменять правила блокировки;
- применять правило ко всем клиентам или только к выбранным;
- блокировать домены, IPv4-адреса и сети CIDR;
- ограничивать правило протоколом TCP или UDP;
- указывать отдельные порты и диапазоны портов;
- включать, отключать и менять порядок правил;
- применять подготовленные правила на сервере.

Правила выполняются сверху вниз.

### Cascade

- отдельный мастер схемы `Client → Entry → Exit → Internet`;
- создание служебного AWG-клиента на Exit;
- импорт и проверка профиля на Entry;
- назначение маршрута выбранным клиентам без ручного редактирования Network;
- проверка AWG outbound и policy routing;
- безопасное отключение с возвратом клиентов на SG-AWG Gateway;
- понятная замена Exit без перевыпуска обычных пользовательских ссылок.

### Cluster и SG-Node

- Controller и записи дополнительных SG-Node;
- схема «две установки + одно подключение»;
- одноразовый enrollment token сроком 30 минут;
- heartbeat, адреса, версии, службы, CPU, память, диск и load average;
- безопасные задания: обновление состояния, диагностика, перезапуск известных служб и синхронизация клиентов из единого раздела Clients;
- отсутствие произвольного удалённого shell;
- единый раздел Clients для Controller и всех SG-Node с выбором сервера при создании;
- стандартный UDP-порт `585` на каждой Node и подтверждение реального `awg0` перед выдачей `.conf`;
- один полный установщик для любого сервера; Agent заранее подготовлен и запускается только после явного подключения к Controller.

### Встроенная Help

- полнотекстовый поиск по справке;
- пошаговые инструкции по всем разделам;
- полные сценарии Cascade и Cluster;
- отключение, замена Exit, повторное подключение Node и типовые ошибки;
- перечень секретов, которые нельзя публиковать в диагностике.

### Доступ к панели

- backend панели слушает только `127.0.0.1:18080`;
- публичный доступ обеспечивает Nginx;
- при установке выбирается отдельный TCP-порт панели;
- поддерживаются HTTP и HTTPS с Let’s Encrypt;
- порт `443` может использоваться страницей-заглушкой, а панель остаётся на выбранном отдельном порту.

### Безопасность

- пароль администратора;
- IP allowlist;
- активные сессии;
- завершение отдельных или остальных сессий;
- журнал успешных, ошибочных и заблокированных попыток входа;
- CSRF-защита форм.

### Обслуживание

- просмотр состояния сервера и служб;
- журналы и диагностический отчёт;
- создание и восстановление резервных копий;
- проверка наличия обновлений и запуск обновления;
- полный uninstall;
- отдельный read-only аудит после удаления.

## Требования

- отдельный сервер или EC2;
- Ubuntu Server `22.04 LTS` или `24.04 LTS`;
- архитектура `amd64`;
- права `root` или запуск через `sudo`;
- TCP `22` для SSH;
- один свободный TCP-порт панели из диапазона `49152–65535`;
- UDP-порт AmneziaWG, по умолчанию `585`;
- для HTTPS — TCP `80`, TCP `443` и выбранный порт панели.

Установщик и полный uninstall рассчитаны на выделенный сервер, где AmneziaWG, Nginx и Certbot не обслуживают другие проекты.

## Рекомендуемая установка на новую EC2 без unzip

Загрузите на сервер один самодостаточный файл:

```text
0.7.0-RC3-INSTALL-SG-AWG-PANEL.run
```

Затем выполните:

```bash
sudo bash 0.7.0-RC3-INSTALL-SG-AWG-PANEL.run
```

Файл `.run` уже содержит полный исходный код панели. `unzip`, GitHub и предварительные команды установки не требуются. Интернет нужен только для системных пакетов Ubuntu и AmneziaWG.

Установщик сначала запрашивает имя сервера, пароль и публичный TCP-порт. После этого технический вывод записывается в журнал, а на экране остаются этапы, зелёная вертушка, время выполнения и понятный итог.

На второй EC2 загрузите и запустите **тот же файл**:

```bash
sudo bash 0.7.0-RC3-INSTALL-SG-AWG-PANEL.run
```

Обе установки изначально самостоятельны. В разделе Cluster выберите, какая панель будет Controller, добавьте второй сервер и выполните показанную одноразовую команду подключения.

## Установка из ZIP, когда unzip уже установлен

```bash
unzip 0.7.0-RC3-AWG-Panel.zip
cd 0.7.0-RC3-AWG-Panel
sudo bash install.sh
```

Действующая установка не перезаписывается. Для обновления существующей панели используется `sudo bash update.sh`.

## Чистая установка из GitHub

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.7.0-RC3/install.sh | sudo bash
```

Установщик запросит:

1. имя этого сервера;
2. пароль администратора;
3. публичный TCP-порт панели, рекомендуется `62443`.

Журнал установки:

```text
/var/log/sg-awg-panel-install.log
```

## Первый вход

После установки откройте:

```text
http://PUBLIC_IP:ВЫБРАННЫЙ_ПОРТ
```

Например:

```text
http://203.0.113.10:62443
```

После входа открывается:

```text
System → Resources
```

## Включение HTTPS

1. Создайте DNS-запись `A`, направленную на публичный IP сервера.
2. Разрешите TCP `80`, TCP `443` и выбранный TCP-порт панели.
3. Откройте `Security → Panel Access`.
4. Выберите `HTTPS + Let's Encrypt`.
5. Укажите домен без `http://` и `https://`.
6. Проверьте порт панели.
7. Нажмите **Сохранить и применить**.

После успешного применения:

```text
https://YOUR_DOMAIN:ВЫБРАННЫЙ_ПОРТ
```

## Основные разделы панели

| Раздел | Назначение |
|---|---|
| **System** | ресурсы, состояние служб, журналы и диагностика |
| **Clients** | создание клиентов и выдача конфигураций |
| **AWG Server** | параметры сервера и конфигурация |
| **Network** | Traffic Rules, DNS и отдельные AWG Outbounds |
| **Cascade** | схема Entry → Exit и назначение клиентов |
| **Cluster** | Controller, SG-Node, heartbeat, метрики и задания |
| **Security** | доступ к панели, HTTPS, пароль, IP allowlist и сессии |
| **Maintenance** | резервные копии и обновление |
| **Help** | полная встроенная пошаговая справка |


## JSON и System Files

Редактируемый JSON предназначен для опытных пользователей и работает с теми же настройками, что и обычные формы.

`System Files` показывает фактические активные файлы AmneziaWG, Nginx, Traffic Rules и systemd, созданные панелью. Раздел доступен только для чтения, копирования и скачивания; редактировать эти файлы вручную не следует.

## Обновление

Для обновления из полного ZIP-архива распакуйте новую версию в отдельную папку и запустите настоящий обновлятор:

```bash
unzip 210-AWG-Panel.zip
cd 210-AWG-Panel
sudo bash update.sh
```

`update.sh` не запускает чистую установку: он создаёт резервную копию, сохраняет Clients, ключи, Cluster, Cascade и настройки, обновляет только код панели, проверяет интерфейс и автоматически откатывается при ошибке.

Обновление из интерфейса также доступно в:

```text
Maintenance → Updates
```

После обновления проверьте версию панели, состояние AWG Server, Traffic Rules и подключение клиента.

## Быстрая проверка

```bash
systemctl is-active sg-awg-panel.service
systemctl is-active sg-awg-server.service
systemctl is-active sg-awg-traffic.service
systemctl is-active nginx.service
curl -fsS http://127.0.0.1:18080/health
```

Ожидается состояние `active` и ответ health check без ошибки.

## Полное удаление

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.7.0-RC3/uninstall.sh | sudo bash
```

Подтверждение:

```text
DELETE SG-AWG-PANEL COMPLETELY
```

После удаления:

```bash
sudo reboot
```

Read-only аудит:

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.7.0-RC3/verify-uninstall.sh | sudo bash
```

## Состояние 206

База 201 была установлена на Controller и отдельную SG-Node: подключение Agent, heartbeat, службы и метрики подтвердились. Живой тест выявил, что отдельный «первый профиль» Node создавал конфигурацию с портом 64441, не совпадающую с реально работающим `awg0` на порту 585.

В 202 эта схема удалена. Все клиенты Controller и SG-Node перенесены в единый раздел Clients, а Agent синхронизирует peer в существующий `awg0` выбранной Node.

В 203 добавлены страны серверов и Cascade на общем реестре Cluster. В 204 Unicode-флаги заменены настоящими встроенными SVG, а Cascade получил второй простой сценарий для двух самостоятельных SG-AWG-Panel без Cluster. В 205 исправлен обратный трафик Cascade, добавлена честная раздельная проверка серверного маршрута и клиента, улучшены Clients и переименование сервера. В 206 полный uninstall дополнительно удаляет Agent, токены, состояние и все сетевые следы подключения SG-Node, чтобы повторный тест начинался действительно с чистого сервера.

## Ответственность

Используйте проект в соответствии с законодательством вашей страны и правилами провайдера. Перед установкой на сервер с другими сервисами изучите действия установщика, updater и uninstall-скрипта.
