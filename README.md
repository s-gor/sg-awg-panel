# SG-AWG-Panel

**Система развёртывания и обслуживания сервера на базе движка AmneziaWG 2.0 с собственной веб-панелью управления.**

![Version](https://img.shields.io/badge/version-v0.1.0--rc4-3b82f6)
![Ubuntu](https://img.shields.io/badge/Ubuntu-22.04%20%7C%2024.04-E95420?logo=ubuntu&logoColor=white)
![Python](https://img.shields.io/badge/Python-3.10%2B-3776AB?logo=python&logoColor=white)
![AmneziaWG](https://img.shields.io/badge/AmneziaWG-2.0-2563EB)

Универсальные VPN-панели обычно объединяют множество протоколов, транспортов и сценариев. Это расширяет их возможности, но одновременно усложняет настройку, обслуживание и диагностику.

**SG-AWG-Panel решает одну конкретную задачу:** разворачивает на чистом сервере Ubuntu нативный AmneziaWG 2.0 и предоставляет собственную веб-панель для управления сервером, клиентами, Traffic Rules, безопасностью, резервными копиями и обновлениями.

Это не просто графическая оболочка над готовой конфигурацией. SG-AWG-Panel устанавливает необходимые компоненты, создаёт и применяет настройки сервера и помогает обслуживать всю установленную систему через единый интерфейс.

```text
Клиент AmneziaWG → AWG Server → Интернет
```

Пользовательский трафик проходит через AmneziaWG напрямую. Для работы соединения не требуются Xray, VLESS, Reality или отдельная прокси-цепочка.

> Текущая версия: `v0.1.0-rc4`.

## Что делает SG-AWG-Panel

### AWG Server

- устанавливает AmneziaWG и необходимые системные компоненты;
- создаёт и применяет конфигурацию сервера;
- позволяет изменять Endpoint, UDP-порт, внутреннюю сеть, DNS, MTU и параметры AmneziaWG;
- показывает состояние сервера;
- позволяет запускать, останавливать и перезапускать AWG Server.

### Clients

- создаёт отдельного клиента и пару ключей для каждого устройства;
- назначает клиенту внутренний IP;
- позволяет включать, отключать, изменять и удалять клиентов;
- поддерживает срок действия клиента;
- показывает полный текст конфигурации;
- создаёт QR-код;
- позволяет копировать конфигурацию и скачивать файл `.conf`;
- поддерживает персональную защищённую ссылку на конфигурацию.

### Traffic Rules

Traffic Rules — полноценная рабочая возможность SG-AWG-Panel для гибкой блокировки трафика клиентов.

Панель позволяет:

- создавать и изменять правила блокировки;
- применять правило ко всем клиентам или только к выбранным;
- блокировать домены, IPv4-адреса и сети CIDR;
- ограничивать правило протоколом TCP или UDP;
- указывать отдельные порты и диапазоны портов;
- включать, отключать и менять порядок правил;
- применять подготовленные правила на сервере.

Правила выполняются сверху вниз.

### Доступ к панели

- backend панели слушает только `127.0.0.1:18080`;
- публичный доступ обеспечивает Nginx;
- при установке выбирается отдельный TCP-порт панели;
- поддерживаются HTTP и HTTPS с Let’s Encrypt;
- порт `443` может использоваться страницей-заглушкой, а панель остаётся на выбранном отдельном порту.

### Безопасность

- пароль администратора;
- IP allowlist;
- активные сессии;
- завершение отдельных или остальных сессий;
- журнал успешных, ошибочных и заблокированных попыток входа;
- CSRF-защита форм.

### Обслуживание

- просмотр состояния сервера и служб;
- журналы и диагностический отчёт;
- создание и восстановление резервных копий;
- проверка наличия обновлений и запуск обновления;
- полный uninstall;
- отдельный read-only аудит после удаления.

## Требования

- отдельный сервер или EC2;
- Ubuntu Server `22.04 LTS` или `24.04 LTS`;
- архитектура `amd64`;
- права `root` или запуск через `sudo`;
- TCP `22` для SSH;
- один свободный TCP-порт панели из диапазона `49152–65535`;
- UDP-порт AmneziaWG, по умолчанию `585`;
- для HTTPS — TCP `80`, TCP `443` и выбранный порт панели.

Установщик и полный uninstall рассчитаны на выделенный сервер, где AmneziaWG, Nginx и Certbot не обслуживают другие проекты.

## Чистая установка

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.1.0-rc4/install.sh | sudo bash
```

Установщик запросит:

1. пароль администратора;
2. публичный TCP-порт панели, рекомендуется `62443`.

Журнал установки:

```text
/var/log/sg-awg-panel-install.log
```

## Первый вход

После установки откройте:

```text
http://PUBLIC_IP:ВЫБРАННЫЙ_ПОРТ
```

Например:

```text
http://203.0.113.10:62443
```

После входа открывается:

```text
System → Resources
```

## Включение HTTPS

1. Создайте DNS-запись `A`, направленную на публичный IP сервера.
2. Разрешите TCP `80`, TCP `443` и выбранный TCP-порт панели.
3. Откройте `Security → Panel Access`.
4. Выберите `HTTPS + Let's Encrypt`.
5. Укажите домен без `http://` и `https://`.
6. Проверьте порт панели.
7. Нажмите **Сохранить и применить**.

После успешного применения:

```text
https://YOUR_DOMAIN:ВЫБРАННЫЙ_ПОРТ
```

## Основные разделы панели

| Раздел | Назначение |
|---|---|
| **System** | ресурсы, состояние служб, журналы и диагностика |
| **Clients** | создание клиентов и выдача конфигураций |
| **AWG Server** | параметры сервера и конфигурация |
| **Network** | полноценное управление Traffic Rules для блокировки трафика |
| **Security** | доступ к панели, HTTPS, пароль, IP allowlist и сессии |
| **Maintenance** | резервные копии и обновление |


## JSON и Generated Config

Редактируемый JSON предназначен для опытных пользователей и работает с теми же настройками, что и обычные формы.

`Generated Config` используется только для просмотра создаваемых панелью конфигурационных файлов. Эти файлы не следует редактировать вручную.

## Обновление

Обновление запускается в:

```text
Maintenance → Updates
```

Ручной запуск:

```bash
sudo env SG_AWG_PANEL_VERSION=v0.1.0-rc4   bash /opt/sg-awg-panel/deploy/update-from-github.sh
```

После обновления проверьте версию панели, состояние AWG Server, Traffic Rules и подключение клиента.

## Быстрая проверка

```bash
systemctl is-active sg-awg-panel.service
systemctl is-active sg-awg-server.service
systemctl is-active sg-awg-traffic.service
systemctl is-active nginx.service
curl -fsS http://127.0.0.1:18080/health
```

Ожидается состояние `active` и ответ health check без ошибки.

## Полное удаление

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.1.0-rc4/uninstall.sh | sudo bash
```

Подтверждение:

```text
DELETE SG-AWG-PANEL COMPLETELY
```

После удаления:

```bash
sudo reboot
```

Read-only аудит:

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.1.0-rc4/verify-uninstall.sh | sudo bash
```

## Проверенное состояние RC4

На живом сервере проверены:

- чистая установка;
- вход и работа основных страниц панели;
- создание клиента;
- подключение клиента через AmneziaWG;
- блокировка трафика через Traffic Rules;
- полный uninstall;
- освобождение backend-порта после удаления.

Функции, не перечисленные в этом разделе, не следует считать подтверждёнными только на основании наличия кода или страницы интерфейса.

## Ответственность

Используйте проект в соответствии с законодательством вашей страны и правилами провайдера. Перед установкой на сервер с другими сервисами изучите действия установщика, updater и uninstall-скрипта.
