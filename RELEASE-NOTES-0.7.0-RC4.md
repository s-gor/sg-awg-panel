# SG-AWG-Panel 0.7.0-RC4

Дата: 2026-07-15

`0.7.0-RC4` объединяет последние исправления RC3 в один чистый релиз и добавляет полноценное управление панелью через SSH.

## Единая модель установки

- один полный установщик `0.7.0-RC4-INSTALL-SG-AWG-PANEL.run` используется на любой новой EC2;
- каждая установка сначала остаётся самостоятельной полной панелью;
- Agent заранее подготовлен, но запускается только после явного подключения сервера к выбранному Controller;
- отдельного пользовательского `INSTALL-SG-AWG-NODE.run` нет;
- один updater используется на Controller и SG-Node.

## Новое SSH-меню

Команда:

```bash
sudo sg-awg-panel
```

открывает нумерованное меню управления сервером. Через него доступны:

- состояние и адрес панели;
- перезапуск Panel, AWG, Nginx и Agent;
- полная диагностика;
- смена пароля администратора;
- сброс браузерных сессий и CSRF;
- восстановление доступа;
- проверка клиентов, Cluster, SG-Node и Cascade;
- последние ошибки служб;
- backup и restore;
- переименование сервера;
- проверка и запуск обновления;
- полный uninstall.

Пункт `6` меняет пароль, `9` проверяет клиентов, `10` — Cluster/SG-Node, `11` — Cascade.

## Исправление CSRF и браузерных сессий

- постоянный `AWGPANEL_SECRET_KEY` создаётся при установке и не меняется при каждом запуске;
- каждая панель получает собственное имя cookie;
- две панели, открытые в одном браузере, не используют одну и ту же сессию;
- устаревший CSRF больше не приводит к необработанной странице `Bad Request`;
- пользователь безопасно возвращается на свежую страницу входа;
- HTML входа не кэшируется;
- доступ можно восстановить через `sudo sg-awg-panel repair-access`.

## Клиенты SG-Node

- работающий `awg0` на аутентифицированной Node является источником истины для серверного публичного ключа;
- Controller принимает подтверждённый ключ Agent и обновляет runtime до выдачи профиля;
- enrollment сразу передаёт реальный `awg_runtime`;
- Node Agent передаёт фактические `AllowedIPs` всех peers;
- Controller не выдаёт адрес, уже занятый локальным peer Node;
- старый конфликтующий managed-клиент переносится на свободный `/32`;
- Agent блокирует конфигурацию с повторяющимся VPN-адресом;
- локальные peers Node сохраняются при синхронизации;
- профиль удалённого клиента имеет обычный формат SG-AWG-Panel.

## Cluster, Clients и Cascade

- команда подключения Node появляется сразу после добавления;
- одинаковое имя Node не создаёт новый дубль;
- старые дубли скрываются, предпочтение отдаётся работающей записи;
- Clients использует компактные полезные колонки без обязательной широкой прокрутки;
- отображаются сервер подключения, маршрут, handshake и трафик;
- Direct ↔ Cascade применяется с точечной очисткой старых соединений выбранного клиента;
- результат проверки Cascade остаётся возле кнопки;
- серверный маршрут и фактическое соединение клиента проверяются отдельно;
- полный внешний Cascade очищается понятными действиями на обеих панелях.

## Установка и обслуживание

- самодостаточный `.run` не требует `unzip`;
- зелёная вертушка показывается с начала подготовки;
- технический вывод записывается в журнал;
- updater сохраняет базу, ключи, Clients, Cluster, Cascade и подключение Agent;
- при ошибке updater выполняет автоматический откат;
- полный uninstall доступен из SSH-меню и одной командой с GitHub;
- GitHub Actions использует временные пути и не обращается к `/var/lib/sg-awg-panel`.

## Интерфейс

- сохранён классический тёмный дизайн;
- встроенные модальные окна вместо browser `confirm`, `alert`, `prompt`;
- 64 настоящих SVG-флага;
- WARP не добавлялся;
- подготовлена структура `docs/screenshots/` для GitHub-скриншотов RC4.

## Установка

На обе EC2 запускается один и тот же файл:

```bash
sudo bash 0.7.0-RC4-INSTALL-SG-AWG-PANEL.run
```

## Обновление

```bash
sudo bash 0.7.0-RC4-UPDATE-SG-AWG-PANEL.run
```

## Полное удаление

```bash
sudo sg-awg-panel uninstall
```

или:

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.7.0-RC4/uninstall.sh | sudo bash
```
