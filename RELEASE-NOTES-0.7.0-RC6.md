# SG-AWG-Panel 0.7.0-RC6

RC6 объединяет все исправления и доработки RC5 Build Fix 1–8 в один новый Release Candidate. Это первая версия после RC5, которую установленная панель определяет как доступное обновление через **Maintenance → Updates** и пункт обновления SSH-меню.

## Cluster и SG-Node

Controller и до 12 SG-Node используют отдельные постоянные пулы.

- Controller использует пул `10.77.0.0/24`.
- Поддерживается до **12 SG-Node** с постоянными пулами `10.77.1.0/24` … `10.77.12.0/24`.
- Номер и пул закрепляются за Node и не меняются после перезапуска.
- Исправлено удалённое создание клиента на Node через Controller.
- Добавлено действие **«Обновить подключение ноды»**: Controller заново получает runtime, пул, Endpoint, список peers, занятые IP и состояние служб.
- Статус удалённого клиента автоматически обновляется из «Применяется» в «Активен».
- Controller нельзя подключить к самому себе как SG-Node.
- Мастер Cluster полностью адаптирован для Latte Graphite и больше не использует нечитаемые тёмные или полупрозрачные карточки.

## Клиенты и AmneziaVPN

- Обычный полный туннель экспортируется строкой `AllowedIPs = 0.0.0.0/0, ::/0`.
- Исправлена выдача `.conf`, QR и персональной ссылки для клиентов SG-Node.
- Конфигурация удалённого клиента выдаётся после подтверждённого применения на Node.
- Локальные peers Node не удаляются и не перезаписываются при синхронизации.

## Cascade и Outbound

- Исправлен standalone Cascade между двумя самостоятельными панелями.
- Созданная ссылка сохраняется и остаётся видна после обновления страницы до истечения срока.
- Повторное создание заменяет прежний служебный доступ и сразу показывает новую ссылку.
- IPv4 Outbound принимает стандартный профиль с `AllowedIPs = 0.0.0.0/0, ::/0`; служебный `::/0` корректно отбрасывается.
- Вариант «Другой сервер» стал компактнее: основные действия и технические детали помещаются без лишней прокрутки.

## Резервные копии

Веб-панель и SSH используют один формат копии. Добавлено отдельное действие **«Проверить резервную копию»**.

Проверяются:

- безопасная структура путей;
- обязательные файлы и `manifest.json`;
- размер и SHA-256;
- совместимость версии;
- SQLite `PRAGMA integrity_check`;
- JSON, `web.env` и базовая структура `awg0.conf`.

Проверка запускается после создания и перед восстановлением. Повреждённая копия не применяется.

## SSH-меню

Меню сокращено до полезных действий без старых дублей. Доступны состояние, перезапуск служб, пароль и сессии, проверки Clients/Cluster/Cascade, создание/проверка/восстановление копий, переименование, обновление и полный uninstall. Аварийная команда `sudo sg-awg-panel repair-access` сохранена как отдельная CLI-команда.

## Latte Graphite

- Полноценная светлая тема для панели и страницы входа.
- Оставлены два явных режима: **Тёмная** и **Latte Graphite**.
- Палитра: фон `#E3E9EE`, карточки `#EEF2F4`, вложенные поверхности `#D9E2E8`, поля `#EAF0F3`, границы `#AEBCC7`.
- Исправлен контраст диаграмм памяти/диска, success-бейджей, Cluster и Cascade.
- Зелёные success-компоненты используют фон `#D8EADF`, границу `#8DBFA2`, текст `#17623F`.
- UI build: `sgawg070rc6`.

## Установка, обновление и удаление

- Один installer используется для Controller и будущей SG-Node.
- Установка и обновление получают проверенное дерево GitHub main.
- Перед обновлением создаётся страховочная копия; при ошибке выполняется автоматический откат.
- RC5 видит RC6 как новую версию и может запустить обновление из **Maintenance → Updates**.
- Полный uninstall доступен из SSH-меню и напрямую из GitHub.
