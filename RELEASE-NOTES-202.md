# SG-AWG-Panel 202

Дата: 2026-07-12

## Главное изменение

В 202 разделены две простые роли интерфейса:

- **Cluster управляет серверами** — Controller и SG-Node, Agent, heartbeat, службы, ресурсы, диагностика и безопасные действия.
- **Clients управляет клиентами** — все подключения Controller и всех SG-Node создаются, показываются и удаляются в одном месте.

## Единый раздел Clients

- В форме «Добавить клиента» появилось обязательное поле **«Сервер подключения»**.
- Можно выбрать Controller или любую готовую SG-Node.
- В общем списке у каждого клиента явно показаны сервер, публичный адрес и состояние применения.
- Добавлены фильтр по серверу и состояния «Применяется», «Удаляется» и «Ошибка».
- Конфигурация становится доступна только после подтверждения реального peer на SG-Node.
- Устранено расхождение, когда Cluster показывал профиль, а Clients показывал 0 клиентов.

## Обычный AmneziaWG на каждой SG-Node

- Каждая SG-Node использует собственный публичный IP и стандартный UDP-порт AmneziaWG **585**.
- Удалена отдельная схема UDP 64441.
- Удалена отдельная сеть 10.200.x.0/24 для «первого профиля».
- Agent добавляет управляемые peers в уже работающий `awg0`, сохраняя вручную созданные неуправляемые peers.

## Проверяемое применение

Статус «Активен» выставляется только после того, как Agent подтвердил:

- работающий интерфейс `awg0`;
- реальный `ListenPort = 585`;
- публичный ключ и сеть сервера;
- полный ожидаемый список peers.

При несовпадении клиент получает состояние «Ошибка» с понятным объяснением.

## Cluster и понятный сценарий

- Удалён блок «Первый AWG-профиль» из карточки SG-Node.
- После подключения страница автоматически проверяет heartbeat каждые 5 секунд.
- Добавлена крупная кнопка «Проверить подключение».
- Шаг 5 теперь прямо говорит: нажать «Добавить клиента», указать имя и выбрать SG-Node в поле «Сервер подключения».
- В карточке Node оставлены серверные функции и переходы «Добавить клиента» / «Показать клиентов».

## Миграция 201 → 202

- Существующие локальные клиенты Controller сохраняются.
- Таблица старых отдельных профилей Node удаляется.
- Устаревшие незавершённые задания первого профиля отменяются.
- Адрес клиента больше не уникален глобально: разные серверы могут законно использовать одинаковый внутренний адрес, например `10.77.0.2/32`.
