# SG-AWG-Panel 204

Дата: 2026-07-12

## Настоящие изображения флагов

- Unicode-флаги, которые Windows показывал как `DE` и `US`, заменены встроенными SVG-изображениями.
- Флаги не загружаются из интернета и одинаково отображаются в Windows, Linux и мобильных браузерах.
- SVG используются в шапке, Cluster, карточке Node, Clients, Cascade и рядом с выбранным сервером.
- При неизвестной стране показывается встроенный нейтральный глобус.
- В комплект включено уведомление о лицензии Twemoji для графики флагов.

## Два простых варианта Cascade

### Из Cluster

- Controller и SG-Node выбираются из общего списка.
- Панель автоматически создаёт служебный peer, передаёт конфигурацию через Agent и применяет маршрут.
- В интерфейсе явно показаны термины `Сервер подключения (Inbound)` и `Сервер выхода в интернет (Outbound)`.

### Другой самостоятельный сервер

- Полноценную SG-Node подключать не требуется.
- На Outbound-сервере создаётся одноразовая ссылка сроком 30 минут.
- На Inbound-сервере ссылка вставляется в одно поле; панель проверяет её и включает Cascade.
- Пользователь не вводит ключи, сети, peer IP или команды маршрутизации вручную.
- Новая ссылка заменяет прежний standalone-служебный peer.

## Управление и безопасность

- Клиенты по-прежнему создаются только в Clients на Inbound-сервере.
- Новые локальные клиенты автоматически получают уже активный standalone-Cascade.
- Нельзя одновременно включить на локальном Inbound маршруты Cluster и standalone.
- Есть проверка видимого IP и кнопка «Вернуть прямой выход в интернет».
- Служебную ссылку и вложенный приватный ключ панель не сохраняет в cookie-сессии.
- Все серверы используют обычный AmneziaWG UDP-порт 585 на собственном публичном IP.
