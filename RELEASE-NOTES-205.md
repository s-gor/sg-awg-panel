# SG-AWG-Panel 205

Дата: 2026-07-13

## Критическое исправление Cascade

- Исправлен nftables kill-switch: ограничения по mark теперь применяются только к исходному трафику клиентов с интерфейса AWG.
- Ответные пакеты, входящие через служебный Outbound и возвращающиеся клиенту через `awg0`, больше не блокируются.
- Исправлены и fallback guards, и guards правил Traffic Rules.
- Добавлены регрессионные проверки обратного трафика.

## Честная проверка Cascade

- После импорта ссылки панель автоматически проверяет серверный маршрут.
- Статусы разделены на `Настраивается`, `Серверный маршрут готов`, `Cascade работает` и `Ошибка`.
- Зелёный статус появляется только после свежего handshake VPN-клиента и трафика в обе стороны.
- Карточка показывает назначенных клиентов, ожидаемый IP, время проверки сервера и клиента.
- Добавлены отдельные кнопки `Проверить серверный маршрут` и `Проверить подключение клиента`.

## Понятный интерфейс

- Варианты Cascade открываются настоящими ссылками `?mode=cluster` и `?mode=external` и работают без JavaScript.
- Для самостоятельных серверов прямо объяснено, где Outbound, где Inbound и на каком сервере нажимать кнопку.
- Технические сведения перенесены в раскрываемый блок.
- В Clients перераспределена ширина фильтров, таблицы и правой карточки; трафик и даты больше не дробятся некрасиво.
- Вторичные действия клиента собраны в компактное меню.
- В шапке добавлено явное меню `Переименовать сервер`; переименование также доступно в System и Cluster.

Версии: Panel `v205`, UI `sgawg205`, Node Agent `205`.
