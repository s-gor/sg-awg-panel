# SG-AWG-Panel 207

Дата: 2026-07-13

## Встроенные окна вместо browser confirm/alert/prompt

- Удалены все 18 `confirm()`, один `alert()` и один `prompt()`.
- Подтверждения удаления, восстановления, обновления, изменения доступа и сброса Cascade показываются в едином интерфейсе SG-AWG-Panel.
- Ошибка форматирования JSON и ручное копирование IP также используют встроенное окно панели.

## Clients

- Исправлен конфликт CSS, из-за которого HTML-таблица наследовала `display: grid` от старого компонента.
- Таблица занимает всю доступную ширину основной области и корректно работает рядом с правой карточкой клиента.
- Фильтры получили явные подписи, отдельные кнопки «Применить» и «Сбросить» и индикатор активного фильтра.
- Добавлена колонка «Маршрут» с честным Direct/Cascade/Outbound.
- Правая карточка показывает сервер, VPN IP, маршрут, handshake, трафик, срок и комментарий.

## Cascade для двух самостоятельных серверов

- Добавлена понятная последовательность из четырёх этапов: роли, ссылка Outbound, импорт Inbound и две независимые проверки.
- Действие «Отключить Cascade и начать заново» очищает Inbound, возвращает клиентов к Direct и явно сообщает о втором шаге на Outbound.
- На сервере выхода отдельная карточка объясняет удаление служебного peer и показывает кнопку «Удалить служебный доступ Outbound».
- Серверный маршрут и реальное подключение клиента по-прежнему проверяются отдельно.

## Переименование и оформление

- Текущее имя сервера заметнее показано в шапке.
- В Cluster добавлена отдельная карточка текущего Controller с кнопкой переименования.
- Изменение имени сразу синхронизирует локальную запись Controller в базе.
- Встроенные SVG-флаги сохранены.
- WARP в 207 не добавлялся.

Версии: Panel `v207`, UI `sgawg207`, Node Agent `207`.
