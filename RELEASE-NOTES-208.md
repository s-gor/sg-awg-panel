# SG-AWG-Panel 208

Дата: 2026-07-13

## Cluster без скрытых действий

- Мастер подключения сокращён до четырёх прямых шагов: установить Node, ввести имя, выполнить сразу показанную команду, добавить клиента.
- Кнопка **«Добавить и показать команду»** выводит одноразовую команду с токеном прямо под формой; переход в отдельный экран не требуется.
- Команда больше не исчезает из-за автоматического обновления страницы. Пока она видна, панель проверяет статус Node отдельным запросом и переходит дальше только после heartbeat.
- Для ожидающей или недоступной Node кнопка **«Создать команду подключения»** находится прямо в мастере Cluster.
- Термин «карточка SG-Node» удалён из рабочего сценария и справки.
- Повторное имя не создаёт новую запись: панель переиспользует существующую Node и выдаёт новую команду.
- Безопасные пустые дубли, созданные прежними версиями, очищаются автоматически. Подключённые Node, клиенты и Cascade не затрагиваются.

## Cascade переключается без ручного переподключения

- После изменения маршрута Direct/Cascade/Outbound панель точечно удаляет conntrack-сессии только изменённых VPN-клиентов.
- VPN-туннель и подключения остальных клиентов не сбрасываются.
- Пакет `conntrack` добавлен в чистую установку, обновление панели и runtime SG-Node.
- Для Cascade через Cluster такой же точечный сброс выполняется на локальном Controller и удалённой SG-Node.
- Проверка клиента теперь требует не только handshake и трафик, но и активное соединение с правильной route mark Cascade.
- Подсказки больше не требуют отключать и включать VPN-профиль вручную.

## Clients

- Сокращены чрезмерно длинные заголовки: «Сервер», «Статус», «Последняя активность».
- Колонки получили более компактные пропорции.
- Минимальная ширина таблицы уменьшена, чтобы на обычном desktop-размере не появлялась лишняя горизонтальная прокрутка.
- Полноширинная компоновка и правая карточка клиента сохранены.

Версии: Panel `v208`, UI `sgawg208`, Node Agent `208`.
