# SG-AWG-Panel v0.1.0 RC5 Hotfix 2

Сборка исправляет обнаруженные при живом тесте проблемы Cascade и перерабатывает Cluster по проверенной логике SG-Panel RC70.

## Cascade

- мастер теперь показывает пять последовательных этапов и явные состояния `Выполнено / Текущий / Недоступен`;
- кнопка копирования использует Clipboard API и резервный способ, после успеха показывает `✓ Конфигурация скопирована`;
- служебный клиент Exit создаётся строго в единственном экземпляре и может быть явно удалён кнопкой `Отключить роль Exit`;
- служебный туннель получает отдельный адрес `10.254.x.2/32`, не пересекающийся с обычной сетью клиентов;
- Exit добавляет NAT для адреса служебного peer;
- Entry добавляет обратный маршрут к своей клиентской сети в policy table Cascade;
- выбор клиентов теперь является полным набором: снятие отметки действительно возвращает клиента на обычный выход Entry;
- проверка использует маршрут назначенного клиента и отдельно напоминает выполнить финальную проверку с реального устройства;
- отключение Cascade удаляет служебный outbound и возвращает клиентов на обычный SG-AWG Gateway.

## Cluster

- интерфейс приведён к мастеру SG-Panel из пяти шагов;
- добавлена одна самодостаточная команда подготовки чистой Ubuntu, которая при необходимости сама устанавливает `curl`;
- Node runtime загружается непосредственно с Controller, GitHub на Node не требуется;
- при создании карточки вводится только имя, адреса передаёт Agent;
- добавлены явные этапы подключения, heartbeat и развёртывания первого AWG-профиля;
- добавлены сводные карточки: всего серверов, в сети, ожидают подключения, нет связи, профили Node;
- первый профиль SG-Node использует UDP-порт `64441`.

## Клиенты и интерфейс

- добавлены заметные кнопки удаления клиента с подтверждением;
- блок `Идентификация сервера` доступен на любой вкладке System;
- сохранены имя/IP в верхней панели, версия/UI внизу меню и циферблаты ресурсов;
- исправлено сжатие и переполнение карточек Cascade и Cluster.

## Версия

- Panel: `v0.1.0-rc5-hotfix2`
- UI: `sgawg007`

## Важно для проверки

Автоматические тесты и статические проверки не заменяют живой тест двух EC2. После установки необходимо отдельно проверить обычные подключения обоих серверов, затем полный путь `Client → Entry → Exit → Internet`, а после этого — подключение чистой SG-Node.

Импорт имени и распознавание QR как AmneziaWG в текущей сборке SG Client относятся к отдельному фиксу самого SG Client: панель уже передаёт полный AWG-конфиг и метаданные имени.
