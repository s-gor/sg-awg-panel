# Cascade

Cascade направляет трафик клиентов через два SG-AWG-сервера.

```text
Клиент → Сервер подключения (Inbound) → Сервер выхода в интернет (Outbound) → Интернет
```

## Термины

**Сервер подключения (Inbound)** — сервер, к которому подключается VPN-клиент. Его endpoint уже записан в клиентской конфигурации.

**Сервер выхода в интернет (Outbound)** — сервер, через который трафик выходит в интернет. Его публичный IP видят сайты и сервисы.

В Cascade слова Inbound и Outbound обозначают роли серверов. Это не отдельные пользовательские профили.

## Вариант 1: сервер из Cluster

1. Подключите SG-Node в `Cluster`.
2. Откройте `Cascade` и выберите `Из Cluster`.
3. Выберите сервер подключения (Inbound).
4. Выберите другой сервер выхода (Outbound).
5. Нажмите `Включить Cascade`.
6. Дождитесь статуса `Серверный маршрут готов`.
7. Переподключите VPN-клиент, откройте сайт и нажмите `Проверить подключение клиента`.

Controller и любая доступная SG-Node могут использоваться в обеих ролях. Один сервер нельзя выбрать одновременно в двух ролях.

## Вариант 2: две самостоятельные панели без Cluster

На сервере выхода (Outbound):

1. Откройте `Cascade` → `Другой сервер`.
2. Нажмите `Создать ссылку для Cascade`.
3. Скопируйте ссылку. Она действует 30 минут.

На сервере подключения (Inbound):

1. Откройте `Cascade` → `Другой сервер`.
2. Вставьте ссылку.
3. Нажмите `Проверить ссылку и включить Cascade`.
4. Дождитесь статуса `Серверный маршрут готов`.
5. Переподключите VPN-клиент, откройте сайт и нажмите `Проверить подключение клиента`.

Ссылка содержит отдельный служебный AmneziaWG-профиль. Пользователь не вводит ключи, внутренние сети или маршруты вручную. Новая ссылка на Outbound-сервере заменяет предыдущий standalone-доступ.

## Клиенты

Клиенты всегда создаются в `Clients` на сервере подключения (Inbound). Обычные клиентские конфигурации не меняются. Новые клиенты автоматически получают активный маршрут Cascade.

## Отключение Cascade и полный сброс

### Cascade внутри Cluster

В карточке маршрута нажмите `Отключить Cascade и начать заново`. Controller вернёт сервер подключения (Inbound) на прямой выход и удалит управляемый служебный маршрут. После этого можно выбрать другой сервер выхода (Outbound). Клиентов пересоздавать не требуется.

### Две самостоятельные панели

Полный сброс выполняется отдельно на каждой стороне:

1. На сервере подключения (Inbound) нажмите `Отключить Cascade и начать заново`. Панель удалит локальный маршрут Cascade и вернёт клиентов на прямой выход.
2. На сервере выхода (Outbound) откройте `Cascade` → `Другой сервер` и нажмите `Удалить служебный доступ Outbound`. Это удалит отдельный служебный peer, созданный для внешнего Cascade.

После выполнения только первого шага сервер подключения уже работает напрямую, но служебный доступ всё ещё существует на сервере выхода. Полный сброс завершён только после второго шага. Затем можно создать новую ссылку и подключить другой Outbound. Клиентов пересоздавать не требуется.

Каждый сервер использует обычный AmneziaWG UDP-порт 585 на собственном публичном IP.

## Честная индикация

- `Настраивается` — ссылка принята, но серверная проверка ещё не завершена.
- `Серверный маршрут готов` — Outbound и policy routing работают; это ещё не проверка устройства.
- `Cascade работает` — после серверной проверки подтверждены свежий handshake, трафик клиента и правильная метка маршрута Cascade. Переподключать VPN после смены маршрута не требуется.
- `Ошибка` — основной текст показывает простую причину; команды и служебные параметры находятся в раскрываемых технических деталях.

После включения Cascade панель показывает число назначенных клиентов, ожидаемый выходной IP, время проверки сервера и время проверки клиента.
