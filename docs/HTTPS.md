# HTTPS, домен, заглушка и отдельный порт панели

Backend SG-AWG-Panel всегда слушает только:

```text
127.0.0.1:18080
```

Снаружи панель публикует Nginx. TCP `443` для панели не используется.

## Схема по умолчанию

```text
TCP 80      ACME-проверка Let’s Encrypt и HTTP-заглушка
TCP 443     обычная HTTPS-заглушка
TCP 62443   HTTPS SG-AWG-Panel
TCP 18080   backend только на 127.0.0.1
UDP 585     AmneziaWG
```

Рекомендуемый адрес панели:

```text
https://awg.example.com:62443
```

Один сертификат домена используется одновременно для заглушки на `443` и для выбранного HTTPS-порта панели.

## Настройка через панель

Откройте **Settings → Публичный доступ**.

Поля:

- **Режим** — HTTP или HTTPS;
- **Домен панели** — например `awg.example.com`;
- **Публичный TCP-порт панели** — рекомендуется `62443`;
- **Управлять заглушкой на HTTPS 443** — включите на отдельном сервере.

E-mail не требуется. Для выпуска сертификата используется неинтерактивная регистрация ACME без e-mail.

Нажмите **«Сохранить и применить»**. Панель:

1. проверит выбранный порт;
2. установит Nginx и Certbot при необходимости;
3. сохранит backend на `127.0.0.1:18080`;
4. создаст отдельный конфиг панели;
5. создаст заглушку на `80/443`, если это разрешено;
6. получит или повторно использует сертификат домена;
7. выполнит `nginx -t`;
8. применит конфигурацию;
9. автоматически восстановит предыдущие настройки при ошибке.

## Требования для HTTPS

До применения:

1. A-запись домена должна указывать на публичный IPv4 сервера;
2. TCP `80` должен быть открыт для первого выпуска сертификата;
3. TCP `443` должен быть открыт для заглушки;
4. выбранный TCP-порт панели должен быть открыт в cloud firewall.

Порты `22`, `80`, `443`, `585` и внутренний `18080` нельзя выбрать как публичный порт панели.

## Если на сервере уже есть SG-Panel или другой Nginx-проект

Nginx-конфигурации SG-AWG-Panel отдельные:

```text
/etc/nginx/sites-available/sg-awg-panel.conf
/etc/nginx/sites-available/sg-awg-placeholder.conf
```

Скрипт не удаляет чужие сайты и не очищает весь каталог Nginx.

Если другой проект уже обслуживает тот же домен на TCP `443` и сертификат существует:

1. выключите **«Управлять заглушкой на HTTPS 443»**;
2. выберите свободный порт панели, например `62443`;
3. SG-AWG-Panel повторно использует существующий сертификат;
4. заглушка и продление сертификата остаются под управлением другого проекта.

## Ручной вариант

Отдельный сервер, SG-AWG-Panel управляет заглушкой и сертификатом:

```bash
sudo bash /opt/sg-awg-panel/deploy/configure-panel-access.sh \
  --scheme https \
  --domain awg.example.com \
  --port 62443 \
  --manage-placeholder 1
```

Существующая заглушка и сертификат управляются другим проектом:

```bash
sudo bash /opt/sg-awg-panel/deploy/configure-panel-access.sh \
  --scheme https \
  --domain awg.example.com \
  --port 62443 \
  --manage-placeholder 0
```

Для временного HTTP на другом порту:

```bash
sudo bash /opt/sg-awg-panel/deploy/configure-panel-access.sh \
  --scheme http \
  --port 8088
```

## Проверка

```bash
systemctl is-active nginx
nginx -t
ss -ltnp | grep -E ':80|:443|:62443|:18080'
```

Ожидается:

```text
0.0.0.0:80
0.0.0.0:443
0.0.0.0:62443
127.0.0.1:18080
```

Backend `18080` не должен слушать публичный адрес.
