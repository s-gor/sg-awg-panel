# Установка SG-AWG-Panel

## Требования

- Ubuntu Server 22.04 LTS или 24.04 LTS;
- архитектура amd64;
- отдельный сервер или EC2;
- root или sudo;
- свободный UDP-порт AmneziaWG `585`;
- свободный TCP-порт панели из диапазона `49152–65535`, по умолчанию `62443`;
- внутренний TCP `18080` не должен быть занят посторонним процессом.

## Установка прямо из GitHub main

Используйте обычную SSH-сессию: установщик безопасно читает имя сервера, пароль и порт через `/dev/tty`.

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/main/install.sh \
  -o /tmp/install-sg-awg-panel.sh
sudo bash /tmp/install-sg-awg-panel.sh
```

Отдельный Release или тег не требуется. Отдельно скачанный `install.sh` загружает полное текущее дерево `main`, проверяет версию и запускает тот же установщик, который находится в полном архиве.

До изменения системы проверяются TCP `18080`, выбранный публичный TCP-порт и UDP `585`. Если порт занят, установка останавливается до безопасного освобождения порта.

## Самодостаточный installer

На Controller и на каждой будущей SG-Node запускается один и тот же файл:

```bash
sudo bash 0.7.0-RC6-INSTALL-SG-AWG-PANEL.run
```

Каждый сервер сначала устанавливается как самостоятельная панель. Подключение к Controller выполняется позднее из раздела **Cluster**.

## Установка из ZIP

```bash
unzip 0.7.0-RC6-AWG-Panel.zip
cd 0.7.0-RC6-AWG-Panel
sudo bash install.sh
```

Интернет на сервере нужен для установки системных пакетов и Python-зависимостей. Исходники самой панели берутся из ZIP, а не из Release.

## После установки

Откройте:

```text
http://PUBLIC_IP:ВЫБРАННЫЙ_ПОРТ
```

Например:

```text
http://203.0.113.10:62443
```

Журнал установки:

```text
/var/log/sg-awg-panel-install.log
```

## Повторная установка

Действующая панель не перезаписывается чистым installer. Сначала выполните полный uninstall, перезагрузите сервер и только затем повторите установку. Installer удаляет автоматически только безопасно распознанные остатки уже удалённой панели.

Полный installer и uninstall рассчитаны на выделенный сервер. Не используйте их на машине, где Nginx, Certbot или AmneziaWG обслуживают посторонние проекты.
