# Cluster и SG-Node

SG-AWG-Panel использует простую модель:

```text
Controller управляет серверами → Clients управляет клиентами
```

Каждая SG-Node — самостоятельный AmneziaWG-сервер со своим публичным IP и обычным UDP-портом `585`.

## Подключение за четыре шага

### 1. Установить вторую полную SG-AWG-Panel

На обеих EC2 используется один файл `0.7.0-RC3-INSTALL-SG-AWG-PANEL.run`. Каждая установка сначала является самостоятельной панелью с собственным AmneziaWG. Agent уже подготовлен, но запускается только после явного подключения к Controller.

### 2. Добавить имя и получить команду

Введите понятное имя, например `CC2-Node`, и нажмите **Добавить и показать команду**. Одноразовая команда с enrollment token сразу появится под формой и действует 30 минут. IP и порт вручную не вводятся.

Если SG-Node с таким именем уже существует, панель не создаёт дубль. Для ожидающей или недоступной Node она выдаёт новую команду подключения к той же записи.

### 3. Выполнить команду

Скопируйте показанную команду и выполните её на уже установленной SG-Node. После сообщения:

```text
Подключение завершено. Agent active
```

вернитесь в Cluster. Панель автоматически покажет:

- статус **В сети**;
- публичный адрес;
- Agent и ОС;
- состояния служб;
- реальный UDP-порт `585`;
- рабочую сеть и публичный ключ `awg0`;
- актуальный heartbeat.

### 4. Добавить клиента

Откройте **Clients**, нажмите **Добавить клиента** и выберите SG-Node в поле сервера. Клиент появится в общем списке, а Agent добавит его peer в работающий `awg0`.

## Повторное подключение

В мастере Cluster рядом с ожидающей или недоступной SG-Node нажмите **Создать команду подключения**. Команда появляется на той же странице; искать отдельное действие внутри SG-Node не требуется.

Новая команда отзывает старый agent token. Подключённая запись, имя сервера и клиенты сохраняются.

## Защита от дублей

Панель не создаёт вторую SG-Node с тем же именем. Пустые неподключённые дубли, созданные старыми версиями, безопасно удаляются автоматически. Подключённые серверы, клиенты и Cascade не удаляются.

## Диагностика

```bash
systemctl status sg-awg-node-agent.service
journalctl -u sg-awg-node-agent.service -n 100 --no-pager
awg show awg0
```

Heartbeat идёт от Node к адресу Controller. Отдельный входящий управляющий порт на Node не требуется. Для клиентских подключений разрешите на SG-Node входящий `UDP 585`.
## Адреса клиентов и локальные peers

Controller учитывает не только собственную базу, но и фактические `AllowedIPs` работающего `awg0` на SG-Node. Если на сервере уже есть локальный peer с `10.77.0.2/32`, первый клиент Controller получит следующий свободный адрес, например `10.77.0.3/32`.

При обновлении со старой версии конфликтующий managed-клиент может быть автоматически перенесён на свободный адрес. После этого скачайте его актуальную конфигурацию из Controller: старый файл с прежним `Address` использовать нельзя. Node Agent RC3 дополнительно отклоняет конфигурацию, если managed peer всё ещё пересекается с локальным peer.
