# Cluster и SG-Node

SG-AWG-Panel использует модель:

```text
Controller управляет серверами → Clients управляет клиентами
```

Каждая SG-Node — самостоятельный AmneziaWG-сервер со своим публичным IP, UDP-портом `585` и постоянным VPN-пулом.

## Постоянные пулы RC6

```text
Controller   10.77.0.0/24
SG-Node 1    10.77.1.0/24
SG-Node 2    10.77.2.0/24
...
SG-Node 12   10.77.12.0/24
```

На каждом сервере `awg0` получает `.1/24`, а обычные клиенты — `.2–.254/32`. Номер и сеть закрепляются за Node в базе, показываются в Cluster и Clients и не меняются после перезапуска.

После удаления Node её номер не передаётся другой Node автоматически. Это защищает старые конфигурации от незаметного наследования прежнего адресного пространства.

## Подключение за четыре шага

### 1. Установить вторую полную SG-AWG-Panel

На обеих EC2 используется один файл `0.7.0-RC6-INSTALL-SG-AWG-PANEL.run`. Каждая установка сначала является самостоятельной панелью с собственным AmneziaWG. Agent уже подготовлен, но активируется только после явного подключения.

### 2. Добавить имя и получить команду

На Controller откройте **Cluster**, введите понятное имя и нажмите **Добавить и показать команду**. Одноразовая команда с enrollment token появляется сразу и действует 30 минут.

Пул назначается записи Node при её создании. Если свободных никогда не использовавшихся слотов 1–12 больше нет, Controller честно остановит добавление новой Node.

### 3. Выполнить команду на второй панели

Скопируйте команду и выполните её на уже установленной SG-Node. После сообщения:

```text
Подключение завершено. Agent active
```

вернитесь в Cluster. Controller покажет статус, публичный адрес, Agent, ОС, службы, UDP `585`, назначенный VPN-пул, реальную сеть и публичный ключ `awg0`.

При переходе с RC4 Agent переносит `awg0` и локальных клиентов этой Node из старого `10.77.0.0/24` в назначенный пул. Ключи сохраняются, но конфигурации клиентов с изменившимся `Address` нужно скачать заново.

### 4. Добавить клиента

Откройте **Clients**, нажмите **Добавить клиента** и выберите SG-Node. Controller выдаст адрес внутри постоянного пула этой Node, а Agent применит и повторно проверит peer на работающем `awg0`.

## Повторное подключение

Рядом с ожидающей или недоступной SG-Node нажмите **Создать команду подключения**. Новая команда отзывает старый agent token, но сохраняет запись, имя, номер пула и клиентов.

## Диагностика

На SG-Node:

```bash
systemctl status sg-awg-node-agent.service --no-pager -l
journalctl -u sg-awg-node-agent.service -b -n 100 --no-pager
awg show awg0
ip address show awg0
```

Heartbeat идёт от Node к Controller. Отдельный входящий управляющий порт на Node не требуется. Для клиентских подключений разрешите входящий UDP `585`.
