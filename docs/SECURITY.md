# Безопасность панели

## Backend только на loopback

Python backend не слушает публичный интерфейс. Внешний запрос проходит через Nginx:

```text
Браузер → Nginx → 127.0.0.1:18080 → SG-AWG-Panel
```

## Смена пароля

Откройте **Security → Сменить пароль**.

После успешной смены:

- новый хеш записывается в `/etc/sg-awg-panel/web.env`;
- все активные сессии завершаются;
- требуется повторный вход с новым паролем;
- событие появляется в журнале входов.

## Активные сессии

Панель хранит сессии на сервере. Для каждой отображаются:

- IP;
- время создания;
- последняя активность;
- User-Agent;
- признак текущей сессии.

Можно завершить одну сессию или все остальные.

## IP allowlist

Пустой allowlist разрешает административный вход с любого IP.

Можно указать:

```text
203.0.113.10
198.51.100.0/24
2001:db8::/48
```

Записи разделяются запятыми или строками. Одиночный IPv4 автоматически становится `/32`, IPv6 — `/128`.

Панель не позволит сохранить allowlist, в который не входит текущий IP. Персональные ссылки `/a/<token>` не ограничиваются административным allowlist.

## Журнал входов

Хранятся последние 1000 событий, в интерфейсе показываются последние 100:

- `login_success`;
- `login_failed`;
- `login_blocked`;
- `logout`;
- `session_revoked`;
- `password_changed`;
- `allowlist_denied`.

## Защита от перебора

После пяти ошибочных попыток с одного IP вход блокируется на 15 минут. Счётчик находится в памяти процесса и сбрасывается после перезапуска панели; все попытки остаются в журнале SQLite.
