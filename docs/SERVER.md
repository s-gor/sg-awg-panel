# Server

Для первого запуска оставьте:

```text
UDP-порт:       585
Сеть клиентов:  10.77.0.0/24
DNS:            1.1.1.1, 1.0.0.1
MTU:            1280
```

Публичный IPv4 и внешний интерфейс панель пытается определить автоматически. Кнопки **«Определить»** повторно подставляют найденные значения.

## Параметры маскировки

- **Проверенный профиль** возвращает базовые значения `Jc`, `Jmin`, `Jmax`, `S1–S4`, уже использованные в реальном тесте.
- **Новые H1–H4** создаёт четыре непересекающихся диапазона без смены серверных и клиентских ключей.
- `I1–I5` остаются расширенными необязательными параметрами.

Кнопка **«Сохранить и запустить»** или **«Сохранить и перезапустить»** выполняет:

1. резервное копирование базы и `awg0.conf`;
2. проверку введённых значений;
3. создание безопасных диапазонов `H1–H4`, если они пусты;
4. проверку временного конфига через `awg-quick strip`;
5. запись рабочего `awg0.conf`;
6. запуск или перезапуск службы;
7. откат при ошибке.

Проверка:

```bash
systemctl is-active sg-awg-server
ip -br addr show awg0
ss -lunp | grep ':585'
```

`UNKNOWN` у интерфейса `awg0` является нормальным состоянием для WireGuard-подобного интерфейса.
