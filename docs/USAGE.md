# Настройка сервера и клиентов

## Сервер

На главной странице панель пытается автоматически определить:

- публичный IPv4 VPS;
- внешний интерфейс, например `ens5` на EC2.

Проверьте значения и оставьте для первого теста:

```text
UDP-порт:       585
Сеть клиентов:  10.77.0.0/24
DNS:            1.1.1.1, 1.0.0.1
MTU:            1280
```

Параметры `H1–H4` при первой настройке можно оставить пустыми. Панель создаст четыре непересекающихся диапазона автоматически.

Нажмите **«Сохранить и запустить»**. Панель выполнит:

1. резервное копирование текущей базы и `awg0.conf`;
2. проверку введённых параметров;
3. формирование временного файла с допустимым именем для `awg-quick`;
4. запись `/etc/amnezia/amneziawg/awg0.conf`;
5. запуск или перезапуск службы;
6. автоматический откат при ошибке.

Проверка на сервере:

```bash
systemctl is-active sg-awg-server
ip -br addr show awg0
ss -lunp | grep ':585'
```

Ожидается:

```text
active
awg0  UNKNOWN  10.77.0.1/24
```

Статус интерфейса `UNKNOWN` для WireGuard/AmneziaWG является нормальным.

## Клиент

1. Введите название устройства.
2. Нажмите **«Создать клиента»**.
3. Откройте **«Конфиг / QR»**.
4. На Windows нажмите **«Скачать .conf»**.
5. Импортируйте файл в AmneziaVPN через добавление конфигурационного файла.
6. На мобильном клиенте с поддержкой AWG 2.0 можно отсканировать QR-код.

Длинный текст конфигурации вручную копировать не нужно.

Обычный WireGuard и WireGuard-режим v2rayN не подходят: они не понимают параметры AWG 2.0 `Jc`, `S1–S4`, `H1–H4`.

## Управление доступом

- **Отключить** — клиент остаётся в базе, но удаляется из серверного конфига.
- **Включить** — доступ возвращается.
- **Пересоздать ключи** — адрес клиента сохраняется, но старые `.conf` и QR перестают работать.
- **Удалить** — клиент и его ключи удаляются из базы.

Все действия автоматически обновляют конфигурацию и перезапускают AWG, если сервер активен.

## Проверка подключения

После подключения клиента внешний IP должен принадлежать вашему VPS, например AWS EC2.

На панели появятся:

- время последнего handshake;
- принятый трафик RX;
- переданный трафик TX.
