# Руководство пользователя SG-AWG-Panel

Это руководство описывает подтверждённые функции версии `v0.7.0-RC4`.

## Первый вход

После установки откройте:

```text
http://PUBLIC_IP:ВЫБРАННЫЙ_ПОРТ
```

После авторизации открывается `System → Resources`.

## System

Раздел показывает ресурсы сервера, состояние основных служб, журналы и диагностическую информацию.

Для базовой проверки должны работать:

- `sg-awg-panel.service`;
- `sg-awg-server.service`;
- `sg-awg-traffic.service`;
- `nginx.service`;
- health check на `127.0.0.1:18080`.

## Clients

Для каждого устройства создавайте отдельного клиента.

Панель позволяет:

- создать клиента;
- изменить имя, комментарий и срок действия;
- включить или отключить клиента;
- пересоздать ключи;
- удалить клиента;
- показать полный текст конфигурации;
- показать QR-код;
- скопировать конфигурацию;
- скачать `.conf`;
- включить персональную защищённую ссылку.

После пересоздания ключей старый клиентский конфиг перестаёт работать.

## AWG Server

В разделе сервера доступны:

- Endpoint;
- UDP-порт;
- внутренняя сеть;
- DNS;
- MTU;
- внешний интерфейс;
- параметры AmneziaWG;
- запуск, остановка и перезапуск сервера.

После изменения Endpoint, порта или параметров клиента скачайте конфигурацию клиента заново.

## Network → Traffic Rules

Traffic Rules используются для блокировки трафика или направления выбранных доменов/IP через настроенный Outbound. Обычный SG-AWG Gateway остаётся неявным маршрутом по умолчанию.

Правило может содержать:

- всех клиентов или выбранных клиентов;
- домены;
- IPv4-адреса и сети CIDR;
- TCP или UDP;
- отдельные порты или диапазоны портов.

Правила выполняются сверху вниз. После изменения правил проверьте их на одном клиенте, прежде чем применять широкую блокировку.


## Security

Раздел содержит:

- HTTP или HTTPS;
- домен и публичный порт панели;
- пароль администратора;
- IP allowlist;
- активные сессии;
- журнал входов.

Backend панели остаётся на `127.0.0.1:18080`, публичный доступ обеспечивает Nginx.

## Maintenance

В разделе доступны резервные копии и обновление.

После восстановления резервной копии или обновления проверьте:

1. вход в панель;
2. состояние AWG Server;
3. состояние Traffic Rules;
4. список клиентов;
5. подключение реального клиента.


## JSON

JSON-редакторы предназначены для опытных пользователей. Перед сохранением внимательно проверяйте изменённые значения.

`System Files` показывает фактические файлы, созданные панелью и используемые службами. Раздел доступен только для чтения, копирования и скачивания; не редактируйте эти файлы вручную.

## Удаление

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-awg-panel/v0.7.0-RC4/uninstall.sh | sudo bash
```

Для запуска удаления введите:

```text
DELETE SG-AWG-PANEL COMPLETELY
```

После удаления перезагрузите сервер и запустите `verify-uninstall.sh`.

## Cascade

Откройте **Cascade** и следуйте мастеру выбранного варианта. Для двух самостоятельных панелей полный сброс всегда состоит из двух действий: отключить маршрут на сервере подключения (Inbound), затем удалить служебный доступ на сервере выхода (Outbound). Полная инструкция: [CASCADE.md](CASCADE.md).

## Cluster и SG-Node

Установите ту же полную SG-AWG-Panel на втором сервере, откройте **Cluster**, добавьте SG-Node и выполните сразу показанную одноразовую команду подключения. Полная инструкция: [MULTI-NODE.md](MULTI-NODE.md).

## Встроенная справка

Раздел **Help** содержит пошаговые инструкции и поиск по всем функциям панели. Справка по новым функциям не требует перехода на внешний сайт.
